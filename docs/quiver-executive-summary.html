01 · Concepto

Qué es Quiver

Quiver es el equivalente a Laravel Backpack en el ecosistema FastAPI. El developer lo clona una vez por proyecto y obtiene de inmediato un panel de administración completo, un portal de usuario y un sistema de control de acceso basado en roles, sin escribir una sola línea de infraestructura.

A diferencia de los paquetes pip, Quiver entrega el código fuente completo desde el primer día. El equipo es dueño del código y puede modificar cualquier capa sin restricciones. Las actualizaciones del core se gestionan como un merge manual, igual que cualquier otro fork.

Dirigido a: developers con perfil backend (Python / Laravel) y experiencia media en React. Quiver elimina la fricción de arrancar una nueva aplicación interna, entregando autenticación, gestión de usuarios, permisos y CRUD en minutos.

03 · Funcionalidades

Qué incluye el MVP

Autenticación completa

Login, logout, refresh silencioso, recuperación de contraseña. JWT en memoria + cookie HttpOnly.

RBAC integrado

Roles y permisos granulares. Admin por permiso, portal por rol. Matriz de checkboxes en la UI.

CRUD Engine declarativo

Define el modelo, extiende QuiverCRUD y registra. 7 endpoints generados automáticamente.

Dashboard con widgets

StatCard y ChartWidget configurables. Grids responsivos. Sin código frontend necesario.

Gestión de usuarios

CRUD built-in de usuarios admin. Roles asignables. Desactivación sin borrado físico.

Portal de usuario

Zona separada con UserLayout personalizable, acceso por rol y páginas custom con @quiver_page.

Menú lateral dinámico

Filtrado automático por permisos y roles. Sin configuración manual de visibilidad.

Páginas custom

Registra páginas en admin y portal con @quiver_page. El router las inyecta automáticamente.

CLI integrada

quiver create-superuser, quiver db migrate, quiver db rollback. Setup inicial en segundos.

05 · CRUD Engine

De modelo a panel en 10 líneas

class ProductCRUD(QuiverCRUD):
    # Modelo SQLModel del proyecto — sin adaptadores
    model = Product
    route = "products"

    columns = [
        Column("name", sortable=True),
        Column("price", col_type="currency"),
        Column("is_active", col_type="badge"),
    ]
    filters = [TextFilter("name"), SelectFilter("is_active")]
    order_by = "-created_at"

# Un registro, 7 endpoints generados automáticamente
quiver.register(ProductCRUD)

08 · Seguridad

Autenticación y sesiones

Estrategia de tokens

  • Access token JWT en memoria (15 min)
  • Refresh token opaco — cookie HttpOnly
  • Refresco silencioso transparente vía Axios
  • Revocación total al resetear contraseña

Control de acceso

  • Superuser bypasea todos los checks
  • Permisos en payload del JWT — sin queries extra
  • <Can> y <HasRole> solo son UI — seguridad real en backend
  • Desactivar usuario, no borrar — integridad referencial
09 · Quick Start

De cero a panel en 5 minutos

# 1. Clonar y entrar
git clone https://github.com/tu-org/quiver mi-proyecto
cd mi-proyecto/backend

# 2. Instalar y migrar
pip install -e ".[dev]"
quiver db migrate

# 3. Crear superusuario
quiver create-superuser

# 4. Registrar tu primer CRUD
# En main.py:
quiver = QuiverApp(app)
quiver.register(ProductCRUD)

# 5. Arrancar
uvicorn main:app --reload
# → http://localhost:8000/admin
